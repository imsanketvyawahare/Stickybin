Overview

What is meant by API ?

API stands for Application Programming Interface. It is one of the most important part of moder web applications. API's make the communication between two devices very easy. Let us understand with the help of small example. Suppose john goes to hotel because he is hungry. He calls the waiter and gives his order. He don't directly talk with chef, but the waiter acted as communication medium between john and chef. So, here the waiter acted as an API. Thats how api's works. API's were design so that application built on different platform's can communicate with each other. Like if we have React frontend and Django Backend, Then to establish communication between these two dissimilar platforms, we need API.

When we want to establish communication between two different systems, we use common interface for both to talk or communicate. That common interface is API. These communication takes place securely depending upon the authentication used between the two systems. API's also helps in systematically exposing our database, without just entirely giving out all our information. For example, we have Github users api, which takes username as parameter and return all the information of that user, excluding private information like passwords, private repositories etc. Thus, here API helped them to systematically design which information should be exposed and thus api's helps in better web application development.

What is REST API ?

Representational State Transfer Application programming interface also generally called as REST API, is an Architecture of how the API's should be designed. It's not any technology, but it's a style of developing API's. Previously we had SOAP (Simple Object Access protocol) based API's which used to send data in XML format which was heavy and thus it used to consume larger bandwidth of network, as the SOAP responses were heavy. But, REST responses are mostly JSON based, which is very light weight and don't consume large bandwidth making web application performant. Another thing is SOAP can only work in XML whereas REST uses, JSON, Plain text, HTML and XML. In REST there is slight disadvantage as it is not much secure. It is mostly used for publically available API's but it is not good mechanish to be used for private API's.

How Stickybin Api Works

Here, the REST api sends JSON response to you of all the titles, actual sticky note and also the short code. You can use this Data to get all the information available at the stickybin as it is opensource project, and everything is available publically.

You can just hit the api and get the json response with title of notes, along with actual note and the date of creation.

curl http://127.0.0.1:8000/api/v1/beta/

Here is the code by which you can post data to the stickybin api, with the help of three parameter i.e "title", "note","slug".

curl -X POST -d 'title="<your title>"note="<your note>"&slug="<slug code>"' http://127.0.0.1:8000/api/v1/beta/
